# タスク可視化ツール（task-mieru-kun）仕様書

## 1. 目的

チームメンバーが「今どんなタスクを、いつからいつまで、どのくらいの負担で抱えているか」を大まかに可視化するツール。

- メンバーの負担状況を俯瞰し、**負担が高すぎる／低すぎる人を早期に発見**する。
- 発見した結果は **メンバーへのヒアリングのトリガー** として使う（負担値そのものを調整するための運用ではない）。
- 個人のタスク管理ツールとしても使える。

### 設計思想（重要）

- 負担値はあくまで「実態を映す鏡」。**負担が高いからといってタスクの負担値を下げる、といった運用は行わない。**
- 警告は「問題が起きている可能性のあるシグナル」であり、対応は人による対話（ヒアリング・タスク調整・応援要請など）で行う。

## 2. 全体方針 / 非機能要件

| 項目 | 内容 |
|---|---|
| 形態 | **単一HTMLファイル** で動作（外部サーバー・ビルド不要） |
| 実行環境 | モダンブラウザ（Chrome / Edge / Firefox 等）でファイルを開くだけ |
| 依存 | 外部CDN・通信に依存しない。ローカル環境で完結する（オフライン動作） |
| データ保存 | **localStorage を中心に自動保存** + **JSONエクスポート / インポート**（共有・バックアップ用） |
| 言語 | 日本語UI |

> 補足：単一HTMLのため HTML / CSS / JavaScript は1ファイルに内包する。ライブラリを使う場合もインライン同梱とし、外部読み込みは行わない。

## 3. 利用シーン

チーム俯瞰と個人管理を **同等** に扱う。トップレベルでビューを切り替えられる。

- **チームビュー**：全メンバーを縦に並べ、各人の負担・タスク期間を俯瞰する。
- **個人ビュー**：特定メンバー（=自分）に絞り、自分のタスクを管理・確認する。

### 時間軸の考え方（中長期の俯瞰）

本ツールは「大まかな負担感」を **中長期** で把握することを主眼とする（例：年間計画、四半期単位のざっくりした計画づくり）。

- 時間軸の単位は **週**（月曜始まり）。日単位の細かな管理ではなく、週ごとの負担感の増減を見る。
- 表示期間は **四半期（13週）／半年（26週）／1年（52週）** を切り替えられ、前後に移動できる。
- 想定する使い方：まず大枠の負担を置き、随時発生したタスクを入れて「負担が高くなりすぎていないか」「もともと入れていたタスクが終わって想定より負担が減っていないか」を確認する。

## 4. データモデル

### 4.1 Member（メンバー）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意ID |
| name | string | 表示名 |
| capacityWarnHigh | number | 上限警告の閾値（デフォルト 80） |
| capacityWarnLow | number | 下限警告の閾値（デフォルト 60） |
| color | string | 表示色（任意） |

- キャパシティの基準値は **100**。各タスクの負担はこの100に対する割合（%）で表す。
- 警告閾値（`capacityWarnHigh` / `capacityWarnLow`）は **メンバーごとに個別設定** できる。新規メンバー作成時のデフォルトは上限 **80** / 下限 **60**。チーム共通テンプレートのような一括設定は本バージョンでは持たない（スコープ外）。

### 4.2 Task（タスク）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意ID |
| memberId | string | 担当メンバーID |
| title | string | タスク名 |
| load | number | 負担値（キャパシティに対する%）。**100超の値も許容**して保持・表示する |
| startDate | date | 開始日（YYYY-MM-DD） |
| endDate | date | 終了日（YYYY-MM-DD） |
| status | enum | `todo` / `in_progress` / `done`（任意） |
| note | string | メモ（任意） |

### 4.3 負担モデル（最重要）

- **キャパシティ比率モデル** を採用する。各タスクの `load` は「そのメンバーのキャパシティ（=100）の何%を占めるか」を表す。
- タスクの負担はその **期間（startDate〜endDate）の各日に均等にかかる** ものとして扱う。
  - 例：`load=30`、期間が5日間 → その5日間は毎日 30% の負担が乗る。
  - 1つのタスク内に「重い期間・軽い期間」がある場合は、**使用者がタスクを分割して入力する運用** とする（本バージョンでは期間内の負担変動はモデル化しない）。
- ある日の **日次負担 = その日にアクティブなタスクの load の合計**。
- **週ごとの負担 = その週（月曜〜日曜の7日）の日次負担の平均**。アクティブなタスクが無い日は 0 として平均に含める（端数週・部分週も常に 7 で割る）。表示（ガント・推移グラフ）と警告判定はこの週次負担を用いる。
- 合計が **100 を超えてもクランプしない**。超過していること自体が可視化したい情報のため、100超はそのまま表示し、超過分を視覚的に強調する。

> 将来拡張の余地：工数（時間/人日）ベースの入力から日次負担%へ自動換算するモードは、本バージョンでは対象外（スコープ外）とする。

## 5. 機能要件

### 5.1 タスク・メンバー管理（CRUD）

- メンバーの追加・編集・削除。
- タスクの追加・編集・削除（担当者・負担・期間・ステータス）。
- 入力バリデーション（endDate ≥ startDate、load ≥ 0 など）。

### 5.2 可視化：負担×期間（ガントチャート的ビュー）

- 横軸=**週**（月曜始まり）、縦軸=メンバー。各タスクを期間の帯（バー）で表示する。
- バーには負担値を表現する（色の濃淡 or 高さ or ラベル）。
- 「いつからいつまで」「どのくらいの負担か」がひと目でわかること。
- 表示期間（四半期／半年／1年）を切り替えられ、前後に移動できる。現在週はハイライトする。
- 内部のデータモデルは日次で保持し、表示時に週へ集約する（入力精度は維持する）。

### 5.3 可視化：負担の遷移（時系列）

- メンバーごとの **週ごとの平均負担の推移を折れ線グラフ** で表示する（週次=その週の日次負担の平均）。
- 100 のライン（基準）を補助線として描画し、超過した週を強調する。
- チームビューでは複数メンバーを重ねて／並べて比較できる。

### 5.4 警告（負担の高すぎ・低すぎ検知）

- 判定単位は **期間平均**。
  - 対象期間は **現在表示中の期間**（四半期／半年／1年。前後移動に追従する）とし、その期間の **週ごとの負担の平均値** を算出する。
  - 平均が `capacityWarnHigh`（デフォルト 80）を超える → **過負荷警告**。
  - 平均が `capacityWarnLow`（デフォルト 60）を下回る → **低負荷警告**。
- 基準の考え方：キャパシティ満杯（100）ではなく、**余裕がある状態（〜80）をベスト** とみなす。80超で過負荷、60未満で余力過多のシグナルとして扱う。
- **最大（ピーク）負荷の併記**：平均に加えて、対象期間内で最も負担が高かった週の負荷（ピーク）も表示する。平均が適正でも特定の週だけ過負荷になっているケースを見逃さないため。**ピーク週負荷 > `capacityWarnHigh`（上限閾値）** のとき強調表示する（バッジの過負荷/低負荷判定自体は平均値で行う）。
- 警告はメンバー一覧上にバッジ／色で明示する。
- 警告の文言は「ヒアリングを促す」トーンにする（例：「過負荷の可能性あり。状況を確認しましょう」）。
- **警告が出ても負担値は自動変更しない**（§1 設計思想に従う）。

### 5.5 データ保存・共有

- 変更は localStorage に自動保存する。
- 「エクスポート」で全データを **JSONファイルとしてダウンロード**。
- 「インポート」で JSONファイルを読み込み、既存データを置き換え／マージする。
- 想定運用：個人が編集 → JSONエクスポート → チームで共有・取り込み。

## 6. 画面構成（案）

1. **ヘッダー**：ビュー切替（チーム / 個人）、表示期間切替（四半期 / 半年 / 1年）、前後移動・今日、メンバー追加 / タスク追加、エクスポート / インポート。
2. **チームビュー**
   - メンバー一覧（各人の表示期間の平均負担・最大週負荷・警告バッジ）。
   - ガント風の負担×期間チャート。
   - 負担推移の折れ線グラフ。
3. **個人ビュー**
   - 選択メンバーのタスク一覧（編集可能）。
   - そのメンバーの負担推移グラフ・警告（平均・最大）。
4. **メンバー編集モーダル**：名前・警告閾値（上限・下限）・表示色。
5. **タスク編集モーダル**：タイトル・担当・負担・期間・ステータス・メモ。

> 警告判定の対象期間はヘッダーの表示期間切替に連動するため、独立した設定画面は持たない。各メンバーの警告閾値（上限・下限）はメンバー編集モーダルで設定する。

## 7. スコープ外（本バージョンで作らないもの）

- サーバー連携・リアルタイム共同編集（共有は JSON 受け渡しで代替）。
- 工数（時間/人日）ベースの負担入力モード。
- 認証・権限管理。
- タスク間の依存関係・進捗率の自動計算。

## 8. 決定事項まとめ

| 論点 | 決定 |
|---|---|
| 負担モデル | キャパシティ比率（基準100）。**100超もそのまま表示** |
| 時間軸 | **週単位**（月曜始まり）。表示期間は **四半期 / 半年 / 1年** を切替。データは日次で保持し表示時に週へ集約 |
| 警告基準 | **表示中の期間の平均**（週平均）で判定。上限 **80** / 下限 **60**（余裕がある状態をベストとする）。**最大（ピーク）週負荷も併記** |
| 期間内の負担変動 | モデル化しない。均等で割り切り、変動はタスク分割で表現する運用 |
| データ保存 | **localStorage 中心 + JSON入出力** |
| 利用シーン | **チーム俯瞰と個人管理を同等** に扱う。**中長期（年間・四半期）の負担俯瞰** を主眼とする |
