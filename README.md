# タスク見える君（task-mieru-kun）

チームメンバーのタスクと**負担**を、中長期（四半期〜1年）の時間軸でざっくり可視化するツールです。
「誰が・いつからいつまで・どのくらいの負担で」タスクを抱えているかを俯瞰し、**負担が高すぎる／低すぎるメンバーを早期に発見**して、ヒアリングなどの対話につなげることを目的とします。

> **設計思想**: 負担値は「実態を映す鏡」です。**負担が高いからといってタスクの負担値を下げる運用は行いません。** 警告はあくまで対話を促すシグナルとして扱います。

## 特長

- **インストール不要**: [`tasks.html`](tasks.html) をブラウザで開くだけで動作します。サーバー・ビルド・パッケージ管理・外部依存ライブラリは一切ありません。
- **オフライン完結**: データはブラウザの `localStorage`（キー: `task-tool-data-v1`）に自動保存。外部送信は行いません。
- **共有・バックアップ**: JSON のエクスポート / インポートでデータを受け渡しできます。
- **日本語UI**。

## 使い方

`tasks.html` をブラウザ（Chrome / Edge / Firefox 等）で直接開くだけです。

任意の静的サーバーで配信することもできます。

```sh
npx --yes http-server -p 8777 -c-1
```

初回起動時はサンプルデータが投入されます。

## 主な機能

### ビュー切替

- **チームビュー**: 全メンバーを縦に並べ、各人の平均負担・最大週負荷・警告バッジ、ガント風チャート、負担推移グラフを俯瞰します。
- **個人ビュー**: 特定メンバーに絞って、タスク一覧（編集可能）と負担推移グラフを確認します。

### 時間軸

- 単位は**週**（月曜始まり）。
- 表示期間は **四半期（13週）／半年（26週）／1年（52週）** を切り替え可能。前後移動・「今日」ボタンに対応。
- 現在週はハイライト表示されます。

### 可視化

- **ガント風チャート**: 横軸=週・縦軸=メンバー。タスクを期間の帯（バー）で表示し、負担値を色の濃淡で表現します。
- **負担推移グラフ**: メンバーごとの週平均負担の折れ線グラフ。100（キャパシティ基準）のラインを補助線として描画し、超過週を強調します。

### 計画ベースライン（予定 vs 実績）

- 「計画を保存」で、その時点の全タスクのスナップショットを**単一・手動保存**（上書き）します。
- 「計画を重ねる」トグルで、推移グラフに保存済みの計画を**点線・淡色**で重ね描きし、予定と実績の乖離を可視化します。
- 計画線は表示期間を変えても再計算されます。参照線であり、**警告判定には影響しません**。

### 警告（負担の高すぎ・低すぎ検知）

- 判定単位は**現在表示中の期間の週平均負担**。
- 平均が `capacityWarnHigh`（デフォルト **80**）超 → **過負荷警告**、`capacityWarnLow`（デフォルト **60**）未満 → **低負荷警告**。
- 平均に加え、期間内の**最大（ピーク）週負荷**も併記。ピークが上限閾値を超えると強調表示します。
- 警告は「ヒアリングを促す」トーンで表示され、**負担値を自動変更することはありません**。

### データ保存・共有

- 変更は `localStorage` に自動保存。
- 「エクスポート」で全データを JSON ファイルとしてダウンロード。
- 「インポート」で JSON を読み込み、既存データを置き換えます。

## 負担モデル

- **キャパシティ比率モデル**。各タスクの `load` は「そのメンバーのキャパシティ（基準=100）の何%を占めるか」を表します。
- タスクの負担は期間（`startDate`〜`endDate`）の各日に**均等**にかかるものとして扱います。
- **日次負担** = その日にアクティブなタスクの `load` 合計。
- **週次負担** = その週（月曜〜日曜の7日）の日次負担の平均（アクティブなタスクが無い日は 0）。表示・警告判定はこの週次負担を用います。
- **実効終了日** = `done` かつ `completedDate` があれば `completedDate`、それ以外は `endDate`。負担はこの実効終了日まで計上します（早期完了で負担が減り、延長で増える）。
- 合計が 100 を超えても**クランプせず**、超過分を強調表示します（オーバーの可視化が目的）。
- 期間内に重い／軽い区間がある場合は、タスクを分割して入力する運用とします。

## データモデル

状態は単一オブジェクト `data` に保持され、`localStorage` に保存されます。

### Member（メンバー）

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | 一意ID |
| `name` | string | 表示名 |
| `capacityWarnHigh` | number | 上限警告の閾値（デフォルト 80） |
| `capacityWarnLow` | number | 下限警告の閾値（デフォルト 60） |
| `color` | string | 表示色 |

### Task（タスク）

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | 一意ID |
| `memberId` | string | 担当メンバーID |
| `title` | string | タスク名 |
| `load` | number | 負担値（キャパシティに対する%）。100超も許容 |
| `startDate` | date | 開始日（YYYY-MM-DD） |
| `endDate` | date | 終了日（YYYY-MM-DD） |
| `status` | enum | `todo` / `in_progress` / `done` |
| `completedDate` | date | 完了日（`done` のとき有効）。負担の実効終了日として用いる |
| `note` | string | メモ |

### Baseline（計画スナップショット）

`data.baseline` … `{ savedAt, tasks: [...] }` または `null`。単一・手動保存（上書き）。

## ファイル構成

すべて [`tasks.html`](tasks.html) 一枚に収める方針です（外部依存・ビルド工程を持ち込まない）。

- [`tasks.html`](tasks.html) — HTML / CSS / JS をすべて内包した単一ファイル
  - `<style>` — 全UIのCSS（CSS変数でテーマ管理）
  - `<script>` — 状態管理・描画・永続化
- [`spec.md`](spec.md) — 機能・データモデル・画面構成の詳細仕様
- [`CLAUDE.md`](CLAUDE.md) — 開発ガイド（開発ワークフロー・ルール）

## スコープ外

サーバー連携・リアルタイム共同編集、工数（時間/人日）ベースの入力、認証・権限管理、タスク間の依存関係・進捗率の自動計算、計画ベースラインの複数保持・名前付き管理・自動警告は、本バージョンの対象外です。詳細は [`spec.md`](spec.md) 第7章を参照してください。
